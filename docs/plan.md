# **Структура проекта**

**Цель:**
Сервис, который собирает объекты недвижимости (ДомКлик, Циан, Авито), сохраняет данные, отображает их на карте Яндекс и позволяет фильтровать по требованиям.

---

# **Распределение задач по ролям**

---

# **Frontend 1 — UI/UX + Каркас сайта**

**Ответственность: визуальная часть и взаимодействие с API карты**

### Основные задачи:

1. Создать структуру сайта (HTML/CSS)

   * Главная страница
   * Страница карты
   * Панель фильтров (цена, площадь, район, тип объекта)
2. Дизайн UI (минимальный, аккуратный)
3. Вёрстка адаптивного интерфейса
4. Подключение Яндекс.Карт (JS SDK)

   * Отображение карты
   * Кастомные маркеры
   * Обработка кликов по объектам
5. Интеграция с backend API (GET /objects, фильтры)

### Выход:

Полностью готовый интерфейс + рабочая карта с маркерами.

---

# **Frontend 2 — Логика отображения + SPA функционал**

**Ответственность: динамика фронтенда и интеграция**

### Основные задачи:

1. Реализация фильтрации данных на клиенте (JS)
2. Обновление маркеров на карте без перезагрузки
3. Отрисовка карточек объектов при клике на маркер
4. Подключение автокомплита адресов (suggest API)
5. Тестирование интеграций фронта с backend

---

# **Backend 1 — Архитектура + API**

**Ответственность: серверная логика и выдача данных**

### Основные задачи:

1. Инициализация Django проекта
2. Создание моделей:

   * объект недвижимости
   * параметры (цена, площадь, адрес, ссылка, координаты)
3. Реализация API:

   * GET /objects — выдача всех объектов
   * GET /objects?price_min=&price_max=&area_min= — фильтры
4. Подключение базы данных (PostgreSQL было бы круто, а если SQLAlchemy - имба)
5. Админка Django для проверки данных
6. Подготовка документации API

### Выход:

Надёжное backend API, на которое фронт может опираться.

---

# **Backend 2 — Парсинг + Интеграции**

**Ответственность: сбор данных и конвертация их в удобный формат**

### Основные задачи:

1. Реализовать парсеры:

   * Домклик (если есть API или HTML парсинг)
   * ЦИАН (сложнее — нужно аккуратно)
   * Авито (динамический контент → Selenium/Playwright)
2. Нормализация данных (единый формат)
3. Определение координат по адресу через Geocoder API Яндекс
4. Сохранение объектов в БД
5. Система обновления данных (периодический сбор)

### Выход:

База объектов всегда актуальна и содержит координаты для карты.

---

# Итоговая схема работы

```
[Парсер] → собирает данные → [Django] → сохраняет в БД
                                       ↓
                               API выдаёт JSON
                                       ↓
                     [Фронтенд] → Яндекс.Карта → маркеры
```
